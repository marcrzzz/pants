1.0.x Stable Releases

This document describes releases in the 1.0.x series.

NB: Since this was the first stable release of pants, this notes page does not include the complete set of changes that occurred since the previous stable release (there hadn't been one!)

1.0.1 (05/31/2016)

The 1.0.1 release: there were no additional changes since the 1.0.1-rc0 release candidate.

1.0.1-rc0 (05/21/2016)

This release contains a few performance and behaviour fixes since the 1.0.0 release.

API Changes

  • Add 'transitive' and 'scope' attributes to export of target RB #3845
  • Upgrade to pex 1.1.6. RB #3768

Bugfixes

  • Make pinger.py work with both HTTP and HTTPS. RB #3904
  • Addresses should not equal things that are not addresses. RB #3791

Refactoring, Improvements, and Tooling

  • Split release notes by release branch RB #3890 RB #3907
  • Use a set for the contains check in topo order path for invalidation RB #3786
  • Scrooge gen: Cache resolved scrooge deps RB #3790

1.0.0 (05/01/2016)

The 1.0.0 release! The 1.0.0-rc3 release candidate was accepted without any additional changes.

1.0.0-rc3 (04/26/2016)

Cherry pick fixes for problems found in RC1

API Changes

  • Rename check_published_deps to check-published-deps to be consistent with dashes in goal names RB #3749

Bugfixes

  • startswith() is the correct method for strings, not starts_with() RB #3744

1.0.0-rc1 (04/22/2016)

This release has several changes related to documentation, CI fixes and work in preparation for the 1.0 release.

  • CI work to enable us to move to jenkins
  • Documentation leading up to 1.0
  • Engine work around handling of symlinks
  • Set a global -Xmx default for JVMs
  • improve cache hit rate with eager caching of zinc

  • Fix public API markers based on feedback RB #3442 RB #3718

  • Fix PEP8Error Nit subclass line_range. RB #3714
  • Fix import order issue.
